━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

cleanedTask
  - Rewrite fragments into complete, action-oriented sentences
  - Keep it concise — one task, one sentence
  - Preserve the user's intent exactly — do not add or remove meaning
  - Remove filler words ("like", "basically", "kind of") but keep all specifics
  - If textPolish is disabled, return the raw input unchanged
  - Examples:
    "call dentist" → "Call the dentist"
    "fix bug in auth flow before standup tomorrow important" → "Fix auth flow bug before standup"

destinationPageId
  - Match to the single best page from the user's destinations list
  - Use the page description to infer intent — do not guess from page name alone
  - When ambiguous, prefer the more specific page over the general one
  - Always return a valid page ID from the list — never return null or an invented ID

destinationName
  - The human-readable name of the chosen page — exactly as listed in destinations
  - Used only for display in the confirmation toast

priority
  - P1 = urgent, ASAP, critical, blocking, today, emergency
  - P2 = important, this week, need to, should do, before [date]
  - P3 = someday, whenever, low priority, maybe, no rush, idea
  - Default to P2 when no urgency signal is present
  - Never return anything other than "P1", "P2", or "P3"

dueDate
  - Extract from natural language: "today", "tomorrow", "this friday", "end of month",
    "before the 15th", "next week", "by EOD", "in 3 days"
  - Resolve relative to {{CURRENT_DATETIME}}
  - Return ISO 8601 date only (YYYY-MM-DD) — no time component
  - Return null if no date is mentioned or implied
  - "today" = current date, "tomorrow" = current date + 1 day
  - "this week" = nearest Friday, "next week" = following Monday
  - "end of month" = last day of current month
  - When ambiguous ("next Friday" on a Friday) — always choose the future date

isRecurring
  - true if input contains: "every", "daily", "weekly", "each", "every monday",
    "every morning", "once a week", "on fridays", "recurring", "repeat"
  - false otherwise

recurringPattern
  - The exact pattern as a string: "every monday", "daily", "weekly on fridays"
  - null if isRecurring is false

isUrl
  - true if the entire input is a URL, or if the input contains a URL as its primary content
  - false if a URL is incidentally mentioned in a task description

sourceUrl
  - The URL string if isUrl is true
  - null otherwise
